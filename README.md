# Generative Agent 시뮬레이션

에이전트들이 하루 일정을 직접 정하고, 같은 시간 같은 장소에서 만나면 자동으로 대화합니다. 또한 대화에서 알게 된 사실과 상대에 대한 인상을 기억으로 쌓아가고, 가상의 이틀이 지나면 한 에이전트 입장에서 다른 에이전트들에 대해 알게 된 내용을 리포트로 저장합니다.

LLM은 OpenAI를 기본값으로 설정하였으며, 키가 없는 경우 자동으로 Ollama로 동작하도록 구성했습니다.

## 실행 방법

먼저 가상환경을 생성하고 필요한 패키지를 설치합니다.

    python -m venv venv
    venv\Scripts\activate
    pip install -r requirements.txt

OpenAI를 사용하시려면 `.env.example` 파일을 `.env`로 복사한 뒤 `OPENAI_API_KEY` 값을 입력하시면 됩니다. 키가 없는 경우에는 자동으로 Ollama가 사용되므로, 별도의 설정 없이 그대로 실행하셔도 됩니다.

콘솔에서 자동 시뮬레이션을 실행하면 진행 과정이 출력되고 `report.txt` 파일이 생성됩니다.

    python main.py

웹 화면으로 확인하시려면 아래 명령으로 실행하시면 됩니다.

    streamlit run app.py

## 파일 구성

- `config.py` : 에이전트, 장소, 시간대 등 설정값을 정의합니다.
- `agent.py` : 에이전트 한 명의 정보(프로필, 기억, 관계)를 담당합니다.
- `dialogue.py` : 한 라운드의 대화를 진행하고, 사실을 추출하며, 관계를 갱신합니다.
- `world.py` : 하루 일정을 계획하고, 시간 순서대로 진행하면서 자동으로 대화를 발생시킵니다.
- `report.py` : 리포트를 생성하고 파일로 저장합니다.
- `storage.py` : 시뮬레이션 상태를 json 파일로 저장하거나 불러옵니다.
- `llm.py` : OpenAI 또는 Ollama 호출을 담당합니다.
- `main.py` : 콘솔 실행 진입점입니다.
- `app.py` : Streamlit 화면(수동 대화, 자동 시뮬레이션, 저장/불러오기)을 제공합니다.

## 구현한 내용

- 각 에이전트는 이름, 나이, 직업, 성격을 가지고 시작하며, 초기 기억을 미리 넣어 줄 수도 있습니다.
- 대화가 끝나면 대화 내용에서 각 인물에 대한 사실을 추출하고, 해당 인물을 제외한 나머지 참여자의 기억에 저장합니다.
- 함께 대화한 인물들끼리는 서로에 대한 관계 요약을 갱신합니다.
- 활동 시간대(08시부터 20시)와 장소(학교, 카페, 도서관 등)를 미리 정해 두고, 하루가 시작되기 전에 각 에이전트가 시간대별로 머무를 장소를 스스로 계획합니다.
- 시간 순서대로 진행하면서 같은 장소에 두 명 이상이 모이면 자동으로 대화가 시작됩니다. 다만 집은 개인 공간이므로 집에 있는 에이전트끼리는 만나지 않습니다.
- 가상의 이틀이 지나면 첫 번째 에이전트를 기준으로, 다른 인물들에 대해 알게 된 사실과 관계를 리포트로 저장합니다.

## 설정 변경

`config.py`에서 에이전트(`AGENT_PROFILES`), 시간대(`TIME_SLOTS`), 장소(`PUBLIC_PLACES`), 진행 일수(`NUM_DAYS`) 등을 변경하실 수 있습니다.
